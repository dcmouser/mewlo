Our new asset file replacement mirror directory thing works fine for any files installed under main mewlo/ directory.
But it does not solve the issue of addon packages (mpacks) install through eggs that may be OUTSIDE of the main mewlo directory.
I think we could solve this by having a site create additional mirror directories for each such package using an alias to refer to that mpack's home dir.
But it might be nice to offer a simpler non-config approach to this, by letting user specify a single MIRROR directory, and having mewlo auto map package found to a subdir.
So core/ would be one subdir, and then each other addon would get its own, whether its installed as an egg outside of mewlo main dir or not.
